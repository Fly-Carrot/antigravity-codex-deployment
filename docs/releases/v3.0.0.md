# v3.0.0 Release Notes

v3.0.0 marks the first setup-first public release of Shared Fabric Dashboard.

## Highlights

- one-click shared-fabric storage bootstrap
- workspace-first VSCode enablement for Codex and Gemini
- rich project memory browser in the desktop app
- clickable sync delta records
- exact six-stage task visibility
- macOS app-shell with settings, workspace switching, multi-window support, and setup assistant
- static packaged icon pipeline for the desktop app

## Included in This Release

- canonical shared-fabric bootstrap and sync scripts
- memory expansion and historical backfill tooling
- shared-fabric install templates and doctor chain
- terminal compact dashboard
- desktop dashboard app source and build scripts
- workspace bootstrap entrypoints for Codex and Gemini
- release-ready repository structure for GitHub publishing

## Validation

- `python3 -m unittest discover -s tests -p 'test_*.py'`
- `python3 -m unittest discover -s tools/compact_dashboard/tests -p 'test_*.py'`
- repeated desktop app rebuild and relaunch validation
- icon bundle verification through extracted `icns` inspection
- workspace bootstrap validation for the current repo

## Notes

- The architecture remains canonical-root-first: `global-agent-fabric` stays the single shared state source.
- VSCode integration is workspace-first, not extension-first.
- Historical bridge metadata is still read for compatibility, but the UI now treats it as low-noise provenance rather than a primary dashboard surface.
