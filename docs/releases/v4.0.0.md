# Fabric v4.0.0 Release Notes

Fabric v4.0.0 is the first release that fully repositions the desktop product around an **LLM-native knowledge-base workflow**.

## Highlights

- renamed the visible macOS product from **Shared Fabric Dashboard** to **Fabric**
- reframed the public story so the shared fabric architecture is introduced before the desktop app
- refreshed the public story around a **wiki-first, graph-assisted, terminal-enabled** knowledge-base workflow
- documented Fabric as a governance brain, implementation body, Obsidian knowledge base, and desktop workbench
- wired Maestro into the shared orchestration story for complex subagent delegation
- aligned the product with an **llm-wiki-inspired** architecture:
  - raw sources stay raw
  - the wiki becomes the maintained synthesis layer
  - the graph becomes the navigation layer
  - the terminal becomes the query and maintenance layer
- continued removing acquisition-first messaging from the app surface so Fabric reads as a **processing and knowledge-maintenance tool**
- refreshed the public screenshots around a sanitized demo vault and coherent release narrative
- packaged the desktop app as `Fabric.app` and the public release artifact as `Fabric-v4.0.0-macOS.zip`
- hardened the public starter by disabling default MCP servers, removing machine-local path assumptions, and separating optional acquisition utilities from the core app

## Product Flow

1. **Shared Fabric Setup**
   - establish the canonical multi-agent root, workspace bridge, and runtime contract
2. **Fabric Monitor**
   - inspect runtime health, sync results, and project activity without leaving the app
3. **Obsidian Wiki Foundation**
   - normalize the vault and maintain project wiki structure
4. **Sources + Deep Extraction**
   - turn external raw inputs into normalized source families and semantic knowledge artifacts
5. **Graph + Terminal**
   - navigate the semantic graph and continue retrieval/maintenance work in the embedded terminal

## Included in This Release

- Fabric desktop app source and packaging pipeline
- local vendor JS for wiki and graph rendering
- wiki/graph/source processing prompts for Gemini CLI
- vault normalization, wiki compilation, and semantic graph workflows
- optional acquisition helper scripts under `tools/acquisition/`
- refreshed docs, screenshots, and release packaging

- The public repository is intended to publish under the simple **Fabric** name.
- The bundle identifier remains stable for this release to avoid preference and runtime migration risk.
- The public story is now English-first for 4.0.0.
- Fabric is **inspired by** the llm-wiki pattern; it does not claim endorsement or affiliation.
